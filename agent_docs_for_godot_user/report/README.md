# PDJE Godot User Reports

Use these templates when a Godot game project hits a strange bug, crash,
runtime load failure, input/judge problem, AI problem, or missing wrapper API.

Developers welcome bug reports and feature requests. Agent-generated report
drafts are fine when they make reporting easier, but a human developer must
review the message and send it directly. Share reports in the developer Discord
first:

- https://discord.gg/2Pwju7xhmS

For reproducible bugs, also prepare a related GitHub issue for the relevant
repository after sharing the problem in Discord:

- Core engine: https://github.com/Rliop913/Project-DJ-Engine
- Godot wrapper: https://github.com/Rliop913/PDJE-Godot-Plugin
- Prebuilt plugin package: https://github.com/Rliop913/Project_DJ_Godot

## Templates

- [bug_report.md](bug_report.md): bug report template for crashes, load
  failures, wrong behavior, input/judge issues, AI failures, packaging
  problems, or other bugs.
- [feature_request.md](feature_request.md): feature request template for new
  Godot APIs, workflow improvements, documentation requests, or packaging
  improvements.

Copy the fenced text block from the matching template, fill it in, review it,
then paste it into Discord. Keep private music, database, and model files out
of public channels unless you have the right to share them. Prefer a minimal
substitute project, scene, DB, audio clip, or model when possible. Automated
reporting or unreviewed message sending is discouraged because logs, paths,
assets, and project data can leak personal information or private content.
